### Why
当前仓库内未找到可复用的旧插件代码，需从零重建一个 IDEA 导出插件，用于生成 Seeyon 普通补丁目录和客户 bug jar 补丁目录，替代人工拷贝与手工编译。

目标是让用户在项目树、编辑器、Git 变更列表中，直接完成单文件、单模块、多模块的补丁导出，并在新版 IDEA 中稳定可用。

### What

| 功能 | 验收标准 | 优先级 |
|------|---------|--------|
| 导出普通代码与静态资源 | 任意普通导出结果都以 `seeyon/` 开头；选择 `src/main/java` 下文件时，导出结果写入目标目录的 `seeyon/WEB-INF/classes` 对应路径；选择 `webapp` 下文件时，导出结果写入目标目录的 `/seeyon` 对应路径 | P0 |
| 导出客户 bug jar 目录 | 用户选择 bug jar 导出模式后，产物目录位于 `seeyon/WEB-INF/lib/<jar名>.jar` 下，修改文件直接写入该 jar 目录下的包路径 | P0 |
| 多工程聚合导出 | 在同一 IDEA 项目内，可一次选择多个工程中的文件或 Git 变更项统一导出；导出结果保留各文件所属工程的正确相对路径 | P0 |
| 多入口触发导出 | 项目树右键、编辑器右键、Git 变更列表右键都可触发“导出到 seeyon”；入口打开后执行体验一致 | P0 |
| 编译方式选择 | 导出前可选择编译方式，默认项为“Maven 仅编译当前工程”，可切换为“IDEA 编译”；执行结果在界面中可见 | P0 |
| 导出路径历史记忆 | 用户可选择导出路径；历史路径跨 IDEA 项目共享，最多保留 30 条，重复路径只保留一条，最近使用路径排最前 | P0 |
| 配置缺失时自动恢复 | 用户目录下不存在配置目录或配置文件时，插件自动创建并继续使用，不因空配置报错 | P1 |

**不做的事：** 不做导入、回滚、差异比对；不支持普通代码与客户 bug jar 之外的第三类特殊打包结构自动识别。

### How

#### 1. 正常流程

1. 用户在项目树、编辑器或 Git 变更列表中选择一个或多个文件。
2. 用户点击“导出到 seeyon”。
3. 插件展示导出面板，允许用户选择导出模式、导出路径、编译方式。
4. 默认编译方式为“Maven 仅编译当前工程”；用户也可切换为“IDEA 编译”。
5. 插件按选择内容识别文件类型与所属工程，生成目标目录结构。
6. 插件执行编译（如用户未关闭编译），再复制编译产物或静态文件到目标目录。
7. 插件更新历史导出路径，完成后展示成功结果与输出位置。

#### 场景：导出普通类文件
- **Given** 用户选择了一个或多个 `src/main/java` 下的文件
- **When** 用户执行“导出到 seeyon”并完成导出
- **Then** 产物写入目标目录下的 `seeyon/WEB-INF/classes` 对应路径，且目录结构与包路径一致

#### 场景：导出 webapp 静态资源
- **Given** 用户选择了 `webapp` 下的静态文件
- **When** 用户执行“导出到 seeyon”并完成导出
- **Then** 产物写入目标目录下的 `/seeyon` 对应路径

#### 场景：导出客户 bug jar 补丁
- **Given** 用户选择了客户 bug jar 导出模式并选中了可导出的修改文件
- **When** 用户执行导出
- **Then** 产物目录位于 `seeyon/WEB-INF/lib/<jar名>.jar` 下，修改后的类文件直接写入该 jar 目录下的包路径

#### 场景：聚合导出多个工程的 Git 变更
- **Given** 用户在 Git 变更列表中选中了多个工程的变更文件
- **When** 用户执行导出
- **Then** 插件一次性导出所有选中文件，并为每个文件写入其所属工程的正确目标路径

#### 场景：使用历史导出路径
- **Given** 用户已存在历史导出路径
- **When** 用户再次打开导出面板
- **Then** 面板可选择历史路径，最近一次使用的路径排在最前，重复路径不会出现多条记录

#### 2. 失败处理

| 失败场景 | 处理方式 |
|---------|---------|
| 用户未选择任何可导出文件 | 阻止导出，并提示“未选择可导出文件” |
| 选中内容同时包含不支持的文件类型 | 跳过不支持项，结果中明确列出被跳过文件 |
| Maven 编译失败 | 导出终止，提示当前工程编译失败，并展示可读错误摘要 |
| IDEA 编译失败 | 导出终止，提示 IDEA 编译失败，并展示可读错误摘要 |
| 导出路径不存在 | 自动创建目录后继续导出 |
| 导出路径不可写 | 导出终止，并提示目标目录不可写 |
| 用户选择了 bug jar 导出模式，但当前文件无法映射到该结构 | 导出终止，并提示当前选择不符合 bug jar 导出规则 |
| 部分文件导出成功、部分失败 | 导出结果必须列出成功项、失败项和失败原因 |

#### 3. 边界与约束

| 边界 | 说明 |
|------|------|
| IDEA 版本兼容 | 至少支持 IU-242.23726.103 及以上版本 |
| 运行时兼容 | 支持用户当前 JBR / JDK 21 运行环境 |
| 历史路径范围 | 历史路径为所有 IDEA 项目共享，不按项目隔离 |
| 历史路径上限 | 最多保留 30 条，超出后淘汰最久未使用记录 |
| 历史路径去重 | 同一绝对路径只保留一条记录；再次使用时刷新到首位 |
| 空配置处理 | 配置目录不存在、配置文件不存在时，自动创建并继续使用 |
| 编译默认值 | 默认编译方式必须是 “Maven 仅编译当前工程” |
| 聚合导出范围 | 同一次导出允许包含多个工程，但只处理用户当前显式选中的文件 |
| 静态资源映射 | 仅对 `webapp` 下资源映射到 `/seeyon` |
| 普通代码映射 | 仅对类文件映射到 `seeyon/WEB-INF/classes` |
| bug jar 结构识别 | 仅覆盖用户定义的客户 bug jar 目录结构 |
