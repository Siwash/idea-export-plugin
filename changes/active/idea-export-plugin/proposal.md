### Why（意图）
重做丢失的 IDEA 导出插件，解决 Seeyon 代码补丁与客户 bug jar 导出依赖手工拷贝、易出错、跨模块效率低的问题，并兼容新版 IDEA。

### What（范围）
#### 做什么
- [ ] 支持普通代码导出：类输出到 `classes`，`webapp` 静态资源映射到 `/seeyon`
- [ ] 支持客户 bug jar 导出：按 jar 文件夹名组织，内部写入修改后的 `classes` 路径
- [ ] 支持多工程聚合导出，覆盖项目树、编辑器右键、Git 变更列表右键场景
- [ ] 支持导出路径选择与全局历史记忆：存放用户目录，最多 30 条，去重，LRU，缺失目录自动创建
- [ ] 支持两种编译方式：默认 Maven 仅编译当前工程；可选 IDEA 编译；失败时友好提示

#### 不做什么
- [ ] 不做导入、回滚、差异比对能力（本次仅覆盖导出链路）
- [ ] 不做两类之外的特殊打包结构自动识别（仅覆盖普通代码 / 客户 bug jar）

### 初步技术方向
基于 IntelliJ Platform Action + 持久化状态实现多入口导出，结合模块路径识别、文件映射规则和外部 Maven/IDEA 编译适配完成产物生成。

### 成功标准
- [ ] 在 IU-242 及以上版本可从多入口导出单文件或多模块变更
- [ ] 导出目录结构符合 Seeyon 普通补丁与 bug jar 规范
- [ ] 历史路径跨项目共享，空配置与重复写入不报错
- [ ] 默认 Maven 编译可用，失败信息可读
