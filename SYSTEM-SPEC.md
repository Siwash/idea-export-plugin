## 功能详情

### IDEA 导出 Seeyon 补丁插件

#### 行为描述
该插件支持在 IntelliJ IDEA 的项目树、编辑器右键菜单、Git 变更列表右键菜单中触发“导出到 seeyon”。

插件支持两类导出：

| 功能 | 当前行为 |
|------|------|
| 普通代码与静态资源导出 | `src/main/java` 选中项导出到目标目录下的 `classes` 对应路径；`webapp` 选中项导出到目标目录下的 `/seeyon` 对应路径 |
| 客户 bug jar 导出 | 选中 bug jar 模式时，按模块的 `finalName/artifactId` 组织 jar 文件夹，类文件写入该 jar 目录下的 `classes` 对应路径 |
| 多工程聚合导出 | 同一 IDEA 项目内可一次导出多个工程的选中文件或显式选中的 Git 变更文件 |
| 编译方式 | 支持 `Maven 仅编译当前工程`、`IDEA 编译`，并支持“跳过编译，直接导出已存在产物” |
| 导出结果展示 | 完成后展示输出位置、编译结果、成功/失败/跳过数量，并列出成功项、失败项和失败原因 |
| 路径历史 | 导出路径历史跨项目共享，存储在用户目录，最多 30 条，去重，按最近使用排序 |
| 配置恢复 | 用户目录配置缺失或损坏时自动创建/恢复，不因空配置报错 |

#### 核心流程

1. 用户在项目树、编辑器或 Git 变更列表中显式选择一个或多个文件。
2. 用户触发“导出到 seeyon”。
3. 插件展示导出面板，允许选择导出模式、导出路径、编译方式，或跳过编译。
4. 插件识别文件类型与所属工程，生成普通补丁或 bug jar 的目录规划。
5. 如果未关闭编译，则按所选编译方式执行编译，并解析真实输出目录。
6. 插件复制编译产物或静态资源到目标目录。
7. 插件写回历史导出路径，并展示输出位置与结果明细。

#### 失败处理

| 失败场景 | 当前行为 |
|---------|---------|
| 未选择任何可导出文件 | 阻止导出并提示“未选择可导出文件” |
| 包含不支持的文件类型 | 跳过不支持项，在结果明细中列出被跳过文件 |
| Maven 编译失败 | 终止导出，展示可读错误摘要 |
| IDEA 编译失败 | 终止导出，展示错误摘要与查看 IDEA Build 窗口的指引 |
| 导出路径不存在 | 自动创建后继续导出 |
| 导出路径不可写 | 终止导出并提示目标目录不可写 |
| bug jar 模式无法映射 | 终止导出并提示当前选择不符合 bug jar 导出规则 |
| 部分成功部分失败 | 结果明细同时列出成功项、失败项和失败原因 |

#### 边界与约束

| 边界 | 当前约束 |
|------|------|
| IDEA 版本兼容 | 至少支持 IU-242.23726.103 及以上版本 |
| 运行时兼容 | 使用 JDK 21 运行 |
| 历史路径范围 | 历史路径对所有 IDEA 项目共享 |
| 历史路径上限 | 最多保留 30 条 |
| 历史路径去重 | 同一绝对路径只保留一条，再次使用时刷新到首位 |
| 聚合导出范围 | 仅处理用户当前显式选中的文件 |
| 静态资源映射 | 仅 `webapp` 映射到 `/seeyon` |
| 普通代码映射 | 仅类文件映射到 `classes` |
| bug jar 识别范围 | 仅覆盖普通代码 / 客户 bug jar 两类结构 |
| 非目标 | 不支持导入、回滚、差异比对；不支持第三类特殊打包结构自动识别 |

#### 依赖与关联
- 依赖模块：IntelliJ Platform ActionSystem、VCS DataKeys、Compiler API、用户目录 JSON 配置存储
- 影响模块：插件入口注册、选择解析、编译适配、路径规划、结果通知
